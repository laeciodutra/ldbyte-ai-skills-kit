# 🔍 Frontend Redesign Audit

## Nome
`frontend-redesign-audit`

## Quando Usar
- Antes de redesenhar qualquer interface
- Para avaliar o estado atual da UI antes de melhorias
- Para criar um plano de ação fundamentado

## Objetivo
Auditar visualmente o estado atual da interface, identificar problemas e criar um plano de melhoria priorizado.

## Papel da IA
Você é um **UI/UX Auditor** que avalia interfaces com olho crítico profissional.

## Checklist Antes de Modificar Código
- [ ] NÃO modificar nenhum código nesta fase — apenas auditar
- [ ] Mapear todas as telas/páginas do projeto
- [ ] Identificar o stack e ferramentas de estilo

## Checklist de Execução
- [ ] Auditar hierarquia visual (o que chama atenção primeiro?)
- [ ] Auditar consistência de espaçamento
- [ ] Auditar paleta de cores (consistência e harmonia)
- [ ] Auditar tipografia (fontes, tamanhos, pesos)
- [ ] Auditar botões (estados, consistência)
- [ ] Auditar cards e containers
- [ ] Auditar formulários e inputs
- [ ] Auditar responsividade (320px, 768px, 1440px)
- [ ] Auditar acessibilidade básica (contraste, foco)
- [ ] Auditar dark mode (se existir)
- [ ] Gerar relatório com problemas e plano de ação

## Padrões Visuais Recomendados
(Ver skill `frontend-premium-ui.md` para referência)

## Regras de Segurança
- Não executar scripts desconhecidos durante auditoria

## Regras para Não Quebrar o Projeto
- Esta skill é READ-ONLY — nenhuma modificação deve ser feita

## Stack Compatível
- ✅ Qualquer stack com interface visual

## Prompt Pronto para Usar
```
Audite a interface deste projeto seguindo frontend-redesign-audit.
NÃO modifique nenhum código. Apenas gere um relatório com:
1. Problemas visuais encontrados (com screenshots/referências)
2. Classificação por prioridade (alto/médio/baixo)
3. Plano de ação recomendado
4. Skills sugeridas para cada correção
```

## Resultado Esperado
- Relatório completo de auditoria visual
- Lista priorizada de problemas
- Plano de ação com skills recomendadas

## Critérios de Qualidade
- [ ] Todas as telas foram avaliadas
- [ ] Problemas classificados por prioridade
- [ ] Plano de ação é específico e executável

## Erros Proibidos
- ❌ Modificar código durante auditoria
- ❌ Relatório genérico sem referência ao projeto real

## Como Validar Antes de Finalizar
1. O relatório cobre todas as telas?
2. Cada problema tem prioridade e solução sugerida?
3. O plano de ação é executável?
