# 🔍 Full Project Review

## Nome
`full-project-review`

## Quando Usar
- Ao receber um projeto novo para dar manutenção
- Para avaliação geral antes de começar melhorias
- Para criar roadmap de qualidade

## Objetivo
Revisar completamente o projeto avaliando: código, UI, segurança, performance, acessibilidade e arquitetura.

## Papel da IA
Você é um **Senior Tech Lead** fazendo code review e avaliação técnica completa.

## Checklist de Execução

### Código
- [ ] Estrutura de pastas organizada
- [ ] Nomenclatura consistente
- [ ] Código sem duplicação excessiva
- [ ] Error handling adequado
- [ ] Tipagem (se TypeScript)
- [ ] Dependências atualizadas

### UI/UX
- [ ] Visual consistente e profissional
- [ ] Responsividade
- [ ] Estados de UI (loading, empty, error)
- [ ] Acessibilidade básica

### Segurança
- [ ] Sem credenciais expostas
- [ ] Validação de inputs
- [ ] Proteção contra XSS/CSRF
- [ ] Headers de segurança

### Performance
- [ ] Bundle size razoável
- [ ] Imagens otimizadas
- [ ] Lazy loading implementado

### Resultado
- [ ] Gerar relatório com pontuação por categoria (0-10)
- [ ] Lista de melhorias priorizadas
- [ ] Estimativa de esforço para cada melhoria
- [ ] Roadmap sugerido

## Erros Proibidos
- ❌ Modificar código durante a review
- ❌ Relatório genérico sem referência ao projeto

## Como Validar
1. Relatório cobre todas as categorias
2. Pontuações são justificadas com exemplos
3. Roadmap é específico e executável
